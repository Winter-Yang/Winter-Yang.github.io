初识NSRunloop

NSrunloop

NSRunLoop的主要作用是控制NSRunloop里面线程的执行和休眠,它的本质是一个消息机制的处理模式.在平时接触最多的可能就是NSTimer中使用NSRunloop了，我们先看端代码

-(void)viewDidLoad {

    [super viewDidLoad];
    [self testTimer];
}

-(void)testTimer{
    UIScrollView *myScroll = [[UIScrollView alloc]initWithFrame:[UIScreen mainScreen].bounds];
    myScroll.contentSize = CGSizeMake(375, 10000);
    [self.view addSubview:myScroll];
    myScroll.backgroundColor = [UIColor redColor];
    myScroll.delegate = self;
    [self defalutTimer];//defalult mode 下的timer
    [self commonTimer];//common modes中的timer

}

-(void)defalutTimer {
    [NSTimer scheduledTimerWithTimeInterval:1.0 target:self selector:@selector(doTime) userInfo:nil repeats:YES];
}

-(void)commonTimer {
    NSTimer *timer =[NSTimer timerWithTimeInterval:1.0 target:self selector:@selector(doTime) userInfo:nil repeats:YES];
    [[NSRunLoop currentRunLoop] addTimer:timer forMode:NSRunLoopCommonModes];
}

-(void)doTime {
    NSLog(@"%@",[NSRunLoop currentRunLoop].currentMode);
    NSLog(@"timer--");
}

-(void)scrollViewDidScroll:(UIScrollView *)scrollView {
    NSLog(@"%@",[NSRunLoop currentRunLoop].currentMode);
}

控制台的log如下：

通过log我们可以发现，如果我们滚动scrollView，那么我们会发现，当前的Timer为defalutTimer，commonTimer并没有走，就好像scrollView在滚动的时候将timer暂停了一样。其实就是RunLoop Mode在作怪。

其实在程序开发过程中，我们并不需要收到创建一个runloop，因此在程序进入主线程中默认为我们创建了一个runloop，在我们注册Timer的时候，实际上是在Runloop里面注册了一下，Runloop每次loop时都会检测这个timer，看是否可以触发。，但是当Runloop在A model，timer确注册在B model时，此时runloop是无法检测到这个timer的，所以需要采用commonTimer的方式，在scrollview进行滑动的时候runloop进入UITrackingRunLoopModes模式，因此需要此将timer加入到NSRunLoopCommonModes模式，这样保证scrollview滑动的同时，timer也在同时进行。

这是在开发过程中最常用的地方，当然还有其他使用的地方，比如Table的滑动，多线程等。
